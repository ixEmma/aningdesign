# Firebase Studio vs Codex vs Claude Code: Building the AssetraX Finance App from One Prompt

## Introduction

In this video, I gave three AI coding tools the same prompt to build a personal finance web app called AssetraX. The goal was simple: use the same app idea, use the same prompt, and compare how far each AI tool could take the project.

This is a practical AI coding tools comparison between Firebase Studio, Codex by OpenAI, and Claude Code by Anthropic. Instead of only talking about theory, the video shows what each tool produces when asked to build a real MVP-style finance control app.

AssetraX is a personal finance app idea designed to help users track spending, income, expenses, and finance activity. This video is not a full production build. It is a practical face-off to see which tool gives the best MVP direction, UI, app structure, local workflow, and mobile responsiveness from one prompt.

## The App Idea: AssetraX

AssetraX is a finance control app concept for monitoring personal finance activity. The app idea focuses on helping users track spending, income, expenses, and financial decisions in one place.

The important part of this test is that the app idea stayed the same across all three tools. That makes the comparison more useful because each result comes from the same starting point instead of three different prompts.

## The Prompt Test

Timestamp: 00:35

The test starts with one prompt for building AssetraX. The goal is to see how each AI coding tool interprets the same product direction, user needs, feature expectations, and interface requirements.

This is a useful way to compare AI app builders because the prompt becomes the control. If the same prompt creates very different outcomes, you can start to understand each tool's strengths and limitations.

Here is the focused version of the brief each tool received:

```txt
Build an MVP web app called AssetraX, a personal finance control app.

Include a dashboard with Total Income, Total Expenses, Current Balance,
and Monthly Spending cards. Add a transaction form with a title, amount,
income or expense type, category, and date.

List recent transactions and calculate the balance so income increases it
and expenses reduce it. Show spending by category with simple cards or a
chart-style summary.

Use React, reusable beginner-friendly components, localStorage, and a
structure that can support Firebase later. Keep the interface modern,
minimal, responsive, and suitable for a professional fintech dashboard.
Do not add authentication or overbuild the first version.
```

This preview communicates the product, the essential screens, the calculation rules, the technology, and the limits of the first release. Those details make the results easier to compare because each tool receives the same definition of “done.”

## Testing Claude Code Locally

Timestamp: 10:31

The third test uses Claude Code. The goal is to see how Claude Code handles the same prompt when launching and inspecting the project locally.

## Launching the Claude Code App

Timestamp: 11:30

The video then launches the Claude Code app locally. This matters because a generated app needs to actually run before you can judge it properly.

A tool can produce impressive-looking code, but if the app is hard to launch or inspect, the workflow becomes slower. Local testing helps reveal whether the result is practical.

## Common Beginner Mistakes

- Comparing AI coding tools with different prompts instead of one shared prompt
- Treating the first AI-generated result as a finished production app
- Ignoring mobile responsiveness until the end
- Looking only at visuals and not app structure
- Forgetting to test whether the project actually runs locally
- Skipping review of the generated code and file structure
- Using AI tools without a clear MVP goal
- Forgetting that a finance app needs careful handling before real user data is involved
