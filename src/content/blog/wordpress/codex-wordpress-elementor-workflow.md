## Step-by-Step Guide

### Step 1: Start with a Staging WordPress Website

Timestamp: 0:00

Start with a staging WordPress website instead of experimenting directly on a live client site. A staging site gives you a safe place to test the AI WordPress workflow, import Elementor templates, adjust layouts, and fix problems before anything goes public.

This matters because AI-assisted website work still needs review. Codex can help structure the build, but WordPress designers should keep the workflow inside a controlled environment.

### Step 2: Open the WordPress Dashboard

Timestamp: 1:12

Open the WordPress dashboard and confirm the basic setup is ready. You should know where Elementor is installed, where templates are managed, and where you will import the JSON Elementor template later.

Before using Codex, make sure the WordPress side is ready. A clean dashboard makes the rest of the workflow easier to follow.

### Step 3: Use the GitHub Repo for the Workflow Files

Timestamp: 3:42

Use the GitHub repo as the home for workflow files, prompts, starter structures, and template-related resources. The repo for this workflow is:

[https://github.com/ixEmma/code-wordpress](https://github.com/ixEmma/code-wordpress)

Keeping the files in GitHub makes the workflow easier to repeat. Instead of rebuilding the process from memory, you can return to the same project files and improve them over time.

### Step 4: Choose a Strong Design Reference

Timestamp: 5:39

Choose a design reference before asking AI to build anything. The design reference gives the layout a direction: spacing, section flow, visual hierarchy, content rhythm, and overall style.

Most AI-generated layouts look generic because they start without a strong visual target. A good reference helps Codex understand the design language you want to recreate as an Elementor-ready WordPress template.

### Step 5: Write a Better GPT Prompt for Codex

Timestamp: 7:16

Write a prompt that explains the goal, audience, design reference, page sections, content needs, WordPress constraints, and Elementor output format. A weak prompt usually produces weak website sections.

For this Codex WordPress Elementor workflow, the prompt should tell Codex what kind of page you are building, how the design should feel, what sections are needed, and how the result should support an Elementor JSON template import.

### Step 6: Move the Prompt and Project Context Into Codex

Timestamp: 10:15

Move the prompt, project context, and relevant workflow files into Codex. This gives Codex the information it needs to make decisions that match the website direction.

Codex works best when it has structure. Instead of asking for a random page, give it the design reference, WordPress goal, Elementor constraints, and expected template output.

### Step 7: Let Codex Structure the WordPress Build

Codex helps turn the prompt into a more organized WordPress build plan. It can help define sections, content blocks, layout direction, template structure, and the workflow needed to move from idea to editable Elementor page.

This is where the Codex Elementor workflow becomes useful. The goal is not only to generate code or text. The goal is to organize the build so the final WordPress page is easier to edit.

### Step 8: Use the Starter Template Plugin

Timestamp: 12:05

The starter template plugin helps prepare the WordPress side of the workflow. It gives you a way to bring template structures into WordPress faster instead of manually rebuilding every section from scratch.

If the plugin feels confusing, remember the role it plays: it helps bridge the prepared template workflow and the WordPress website environment.

### Step 9: Apply the Custom Codex Skill

Timestamp: 14:33

The custom Codex skill helps make the workflow repeatable. Instead of writing every instruction from zero each time, the skill can guide Codex toward the correct WordPress, Elementor, and template-building behavior.

Codex skills for WordPress are useful because they preserve workflow knowledge. They help Codex understand what kind of output you need and how the website build should be structured.

### Step 10: Export or Prepare the Elementor JSON Template

After Codex helps structure the build, prepare the Elementor JSON template for import. The goal is to create an Elementor-ready WordPress template that can be brought into the staging website and edited visually.

This step is important because the final template should not be locked away from the designer. It should become editable inside Elementor.

### Step 11: Import the JSON Elementor Template Into WordPress

Timestamp: 16:53

Import the JSON Elementor template into WordPress through the template import workflow. After importing, open the template or page inside Elementor and check that the sections, layout, and content are editable.

Elementor JSON template import can feel confusing at first, but the key is simple: the JSON file carries the page or template structure into WordPress so you can continue editing inside Elementor.

### Step 12: Review the Final Result and Make Design Adjustments

Timestamp: 25:00

After importing, review the final result carefully. Check the design reference, spacing, mobile behavior, section order, typography, button styling, and whether the page still feels editable.

AI can speed up the build, but the designer still needs to make final design adjustments. This is how you use AI without losing control of the design.

## Common Problems and Fixes

### AI-Generated Layouts Looking Too Generic

Use a stronger design reference and explain the visual direction clearly in the prompt. Generic inputs usually produce generic outputs.

### Weak Prompts Producing Weak Website Sections

Add more structure to the prompt. Include the audience, page goal, section list, design reference, WordPress constraints, and Elementor template goal.

### Not Using a Design Reference Before Building

Choose the design reference first. It gives Codex a clearer target and helps the final Elementor-ready template feel more intentional.

### Losing Design Control When Using AI

Keep the workflow designer-led. Use Codex to support structure and speed, then review and adjust the final Elementor page yourself.

### Elementor JSON Template Import Confusion

Make sure you are importing the JSON file through the correct Elementor template area. After import, open the template in Elementor to confirm it is editable.

### Starter Template Plugin Confusion

Understand the plugin as part of the bridge between prepared templates and WordPress. It helps you move faster, but you still need to review the imported page.

### GitHub Repo Files Not Being Used Correctly

Use the repo as a workflow source, not just a storage folder. Keep prompts, references, and template files organized so Codex can use the right context.

### Staging Website Setup Confusion

Work on a staging WordPress website before touching a live project. This keeps testing, importing, and fixing safely separated from production.

### Codex Not Understanding the Build Direction

If Codex does not understand the direction, the prompt probably lacks structure. Add clearer project context, design reference notes, section requirements, and expected Elementor output.
